# Система оптимизации расписания конвейерных систем

**Выпускная квалификационная работа бакалавра**  
Кафедра «Информационные системы», СевГУ, 2024

## Тема

Разработка системы оптимизации расписания выполнения пакетов заданий в конвейерных системах при учёте технического обслуживания приборов с использованием целочисленного программирования.

## Научная база

Основана на статье:  
**Кротов К.В.** Модели смешанного целочисленного линейного программирования оптимизации включения заданий в пакеты и порядков проведения операций с ними в конвейерных системах. *Информационно-управляющие системы*, 2024, № 6, с. 46–57.

**Расширение:** добавлен блок учёта технического обслуживания (ТО) приборов — оригинальный научный вклад ВКР.

## Быстрый старт (Windows)

### Способ 1: Автоматически (рекомендуется)

1. Убедитесь, что установлен **Python 3.10+** (https://python.org)
2. Дважды кликните на файл **`install_and_run.bat`**
3. Подождите установки пакетов (~2-3 минуты при первом запуске)
4. Программа запустится автоматически

### Способ 2: Вручную через командную строку

```cmd
# Открыть cmd в папке с программой (Shift+ПКМ → "Открыть командную строку")

# Создать виртуальное окружение
python -m venv venv

# Активировать
venv\Scripts\activate

# Установить зависимости
pip install PyQt5 matplotlib numpy openpyxl pulp

# Запустить
python main.py
```

### Последующие запуски

Просто дважды кликните **`run.bat`**

## Использование

1. **Вкладка «Параметры задачи»**
   - Введите I (типы), L (приборы), J (позиции)
   - Нажмите «Применить размерность»
   - Заполните матрицы времён обработки, переналадок
   - Включите/выключите учёт ТО, задайте периоды и длительности ТО
   - Выберите критерий: Cmax или G (суммарное запаздывание)
   - Нажмите **«▶ Решить задачу»**

2. **Вкладка «Результаты»** — таблицы составов пакетов, расписания, ТО

3. **Вкладка «Диаграмма Ганта»** — графическое расписание

4. **Вкладка «Анализ»** — серия расчётов, графики зависимостей

5. **Экспорт** — кнопка «Экспорт в Excel» или Файл → Экспорт

## Решатели MILP

| Решатель | Установка | Примечание |
|---|---|---|
| **PuLP (CBC)** | `pip install pulp` | Бесплатный, рекомендуется |
| **IBM CPLEX** | `pip install docplex` + CPLEX CE | Быстрее, но нужна установка CPLEX |

Для задач малого размера (I≤5, L≤5, J≤8) PuLP+CBC работает хорошо.

## Математическая модель

Задача класса **F|batch|Cmax** (FlowShop с пакетной обработкой).

**Переменные:**
- `x[i][j]` ∈ {0,1} — задания i-го типа в j-й позиции
- `m[j]` ∈ ℤ — количество заданий в j-й позиции (≥2)
- `q[l][j]` ≥ 0 — момент начала обработки пакета j на приборе l
- `maint[l][j]` ∈ {0,1} — ТО перед позицией j на приборе l (**новое**)

**Критерии:**
- Модель 1: min Cmax
- Модель 2: min G (суммарное запаздывание)

**Научный вклад:** добавлен блок ТО с ограничениями на суммарное время работы приборов между обслуживаниями.
